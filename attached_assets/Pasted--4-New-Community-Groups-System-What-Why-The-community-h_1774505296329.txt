#4 - New Community Groups System
What & Why
The community has a structured hierarchy of devotee groups that need to be represented in the system. Six new groups must be added with their specific field schemas, plus a general volunteer coordination group (Narayan Bhakt) with dynamically manageable task categories. Dev Mode must allow adding/removing groups and task categories at runtime without code changes.

Done looks like
Shri Golok Dham group is seeded with fields: devotee (linked), first darshan date, test taken by, test date, darshan status, notes; members are promoted devotees who have completed their darshan; this group is visible on the Groups page with its hierarchy badge "Level 3 – Golok Dham"
Braj Basi group exists below Golok Dham (Level 2); contains devotees not yet promoted; fields: devotee (linked), join date, remarks
Darshan Eligibility List — tracks eligible devotees (6+ months in community, eligibility test given); fields: devotee (linked), test taken by, test date, pass/fail, additional remarks, eligibility status
International Working Committee (IWC) — superior to Golok Dham (Level 4); Golok Dham members selected for worldwide representation; fields: devotee (linked), selected date, region/country assigned, role, active status
International Katha Committee — most superior group (Level 5), "The Chosen 17"; fields: devotee (linked), induction date, areas of excellence, milestones achieved, notes
Narayan Bhakt Group — volunteer coordination group; has a dynamic list of task categories (Video Editing, Social Media Publicity, Content Creation, Commenters, etc.); each member entry records: devotee (linked), task category (dropdown from the dynamic list), contribution hours, active status; Dev Mode shows a "Manage Task Categories" panel where categories can be added and removed dynamically, changes immediately reflected in the Narayan Bhakt group entry form
All 6 groups appear on the Groups page with their hierarchy level badge; the Groups page supports sorting/filtering by hierarchy level
Dev Mode "Seed Manager" panel includes a button to re-seed all community groups if accidentally deleted
Each group's "Add Entry" form is pre-configured with the correct fields for that group type; linked devotee fields show a searchable devotee picker
Out of scope
Automated promotion workflows (e.g. auto-move to Golok Dham on criteria met)
External database persistence (in-memory only)
Email/WhatsApp notifications on group changes
Tasks
Group hierarchy field — Add a hierarchyLevel integer field and groupCategory (enum: community, volunteer, committee, eligibility) to the groups schema and in-memory store; update the Groups page to show a hierarchy badge on each group card and allow sorting by level
Seed the 6 new groups — In memoryStorage.ts seed data, add all 6 groups with their complete custom field definitions: Braj Basi (Level 2), Golok Dham (Level 3), IWC (Level 4), International Katha Committee (Level 5), Darshan Eligibility List, and Narayan Bhakt Group; the Narayan Bhakt group stores its task categories as a customizable array in its group record
Narayan Bhakt dynamic task categories — Add a taskCategories JSONB/array field to the Narayan Bhakt group record; add API endpoints to add/remove task categories; the GroupEntryForm for this group renders a dropdown populated from the live task categories list
Dev Mode group management panel — Add a "Group Manager" tab in Dev Studio that lists all groups, allows creating new groups with hierarchy level and custom field wizard, and for the Narayan Bhakt group specifically shows an inline "Task Categories" editor to add/remove categories dynamically
Linked devotee picker in group entry forms — For all 6 new groups, the "devotee" field in GroupEntryForm renders a searchable combobox of all devotees (firstName + lastName + devoteeId), rather than a plain text input
